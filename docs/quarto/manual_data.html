This R script relies on the following helper files:

  • R/Analysis setup.R
  • R/functions.R

This script prepares the manually reviewed dataset for descriptive analysis. It imports the abstraction spreadsheet, filters to confirmed records, creates derived variables from free-text fields, generates a study selection flowchart, and produces the tables used in the manual review results pages. The workflow is shown below in the same sequence used in the analysis script.

Import the manually reviewed data

The analysis begins by importing the reviewer spreadsheet from Excel. This file contains the manually abstracted information for each paper.

Click to show/hide R Code 
Anagha_summer_project_form_2025_raw <- read_excel(
  "data-raw/Data abstraction Excel reviewer 1 and 2.xlsx",
  sheet = "Reviewer 2"
)

Clean the manually reviewed dataset

The imported spreadsheet is standardized using clean_names(), and the analysis is restricted to records that were confirmed for inclusion. This creates the core cleaned dataset used in subsequent steps.

Click to show/hide R Code 
Anagha_summer_project_form_clean <- Anagha_summer_project_form_2025_raw %>%
  clean_names() %>%
  filter(confirmed_by_vh == "Yes")

Create the study selection flowchart

A flowchart is generated from the cleaned dataset to summarize how papers moved through the selection process. The diagram begins with the initial PubMed search results, then filters to new patient-level data analyses and splits the remaining papers into transplant-related and non-transplant-related studies.

Click to show/hide R Code 
Anagha_summer_project_form_clean_fc <- Anagha_summer_project_form_clean %>%
  as_fc(label = "Initial papers from Pubmed search") %>%
  fc_filter(
    is_this_paper_a_new_data_analysis == "Yes",
    label = "New patient-level data analysis",
    show_exc = TRUE
  ) %>%
  fc_split(
    is_this_a_transplant_related_study,
    label = c("Not transplant-related", "Transplant-related")
  )

The resulting flowchart is then drawn and exported as a PNG image for use in the Quarto site.

Click to show/hide R Code 
fc_draw(Anagha_summer_project_form_clean_fc) %>%
  fc_export("Letter analysis/flowchart.png")

Create a table of included papers

The next step is to generate a formatted table listing the papers included in the manual analysis. This table is restricted to new patient-level data analyses and includes the main bibliographic details together with transplant-related status.

Click to show/hide R Code 
paper_table_gt <- Anagha_summer_project_form_clean %>%
  filter(is_this_paper_a_new_data_analysis == "Yes") %>%
  select(
    authors,
    title,
    journal,
    year,
    transplant_related = is_this_a_transplant_related_study
  ) %>%
  distinct() %>%
  arrange(transplant_related, authors, desc(year), title) %>%
  gt() %>%
  cols_label(
    authors = "Authors",
    title = "Title",
    journal = "Journal",
    year = "Year",
    transplant_related = "Transplant related"
  ) %>%
  tab_header(
    title = "Papers Included in the Analysis"
  )

The formatted paper table is then saved for later reuse in the Quarto pages.

Click to show/hide R Code 
paper_table_gt
saveRDS(paper_table_gt, file = "Letter analysis/paper_table_gt.rds")

gtsave(
  filename = "Letter analysis/paper_table_gt.html",
  data = paper_table_gt
)

Create the final dataset for manual analysis

The confirmed dataset is then expanded into a more analysis-ready form. Several free-text fields are converted into binary indicator variables so that the contents of each paper can be summarized systematically. These include programming languages used, USRDS files used, and types of data components abstracted.

    did_the_authors_explicitly_state_what_files_they_used,
    does_the_choice_of_files_codes_seem_appropriate_for_the_research_question,
    files_used_Core,
    files_used_Institution,
    files_used_PhysicianSupplier,
    files_used_Hospital,
    files_used_Transplant,
    files_used_PartD,
    files_used_CROWNWeb,
    component_basicdemographics,
    component_icd9,

    component_cptcode,
    component_medicationD,

      component_icd9,
      component_icd10,
      component_cptcode,
      component_medicationD,
      component_transplant,
      component_costs,
      component_residential
    )
  ) %>%
  add_variable_group_header(
    header = "Files used",
    variables = c(
      files_used_Core,
      files_used_Institution,
      files_used_PhysicianSupplier,
      files_used_Hospital,
      files_used_Transplant,
      files_used_PartD,
      files_used_CROWNWeb
    )
  ) %>%
  add_variable_group_header(
    header = "Programming language used",
    variables = c(
      languages_used_R,
      languages_used_SAS,
      languages_used_Stata,
      languages_used_SPSS
    )
  ) %>%
  modify_footnote_body(
    footnote = "Categories in this section are not mutually exclusive; percentages may not sum to 100%.",
    columns = label,
    rows = label %in% c(
      "Data components abstracted",
      "Files used",
      "Programming language used"
    )
  ) %>%
  as_gt()
