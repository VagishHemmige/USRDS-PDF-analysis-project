LLM review

We used an R-based pipeline built with the ellmer package to evaluate whether large language models (LLMs) could reproduce the manual review process. Each PDF was imported into R, and the full text was submitted to each of three LLMs along with a standardized set of abstraction questions modeled on the manual review form.

The ellmer package was used to request structured outputs, allowing model responses to be returned in a format suitable for downstream analysis in R. This made it possible to compare LLM-generated abstractions directly with human review across multiple study characteristics, including study type, transplant relevance, programming language, files used, and other extracted variables.

Data analysis in R

Data analysis was performed in R. We summarized the results of manual and LLM-based review and compared model outputs with human abstraction.

Agreement between human reviewers and each LLM was assessed using Cohen’s kappa. We also examined simple descriptive summaries of extracted variables, including whether papers represented new data analyses, whether they were transplant-related, which programming languages were reported, and which USRDS files or data elements were used.

To visualize patterns of agreement across variables and models, we generated heatmaps and related summary tables.
